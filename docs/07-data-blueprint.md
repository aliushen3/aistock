# 数据蓝图

## 1. 数据分层

```
L3  应用数据    候选池、报告、批注、审计日志
L2  知识数据    图谱、本体、向量索引、溯源
L1  清洗数据    标准化财报、公告、行业指标
L0  原始数据    PDF、API 原始响应、爬虫快照
```

## 2. 按实体类型的数据源

### 2.1 公司 / 财报

| 字段 | 主数据源 | 备选 | 更新频率 | 质量等级 |
|------|---------|------|---------|---------|
| 营收拆分 | Wind / 同花顺 iFinD | 巨潮公告 | 季报 | A |
| 毛利率 | 财报 | - | 季报 | A |
| 市值 | 行情 API | - | 日 | A |
| PE/PB 分位 | Wind | 自算 | 日 | A |
| 机构覆盖数 | 研报库 | 手动 | 周 | B |

### 2.2 产业链 / 产品

| 字段 | 主数据源 | 备选 | 更新频率 | 质量等级 |
|------|---------|------|---------|---------|
| 上下游关系 | 专家校准 + LLM 抽取 | 研报 | 事件驱动 | B→A |
| 产能 | 研报、行业协会 | 公司公告 | 月 | B |
| 扩产周期 | 专家 + 研报 | - | 季 | C→B |
| CR4 | 行业报告 | 手动 | 半年 | B |
| 认证周期 | 专家 + 招股书 | - | 静态 | C |

### 2.3 非结构化

| 类型 | 来源 | 合规注意 | 更新 |
|------|------|---------|------|
| 券商研报 | 内部订阅 / 合规渠道 | 版权、不得外泄 | 日 |
| 公告 | 巨潮 / 交易所 | 公开 | 实时 |
| 会议纪要 | 买方内部 | 内幕信息隔离 | 事件 |
| 产业舆情 | 新闻 API | 去重、溯源 | 日 |

## 3. 数据源优先级（冲突消解）

```
专家确认 > 公司公告 > 行业协会 > 头部券商研报 > 其他研报 > LLM 抽取 > 默认值
```

## 4. A 股特有问题处理

| 问题 | 处理策略 |
|------|---------|
| ST / *ST | 候选池默认排除，可配置 |
| 借壳 / 重大资产重组 | 标记 `data_quality_warning`，暂停自动提示分 |
| 关联交易 | 供应链关系需人工确认 |
| 信披质量差 | 降低 confidence，要求双源 |
| 北向 / 融资拥挤度 | 作为拥挤度辅助指标 |

## 5. 缺失降级策略

| 缺失字段 | 降级行为 |
|---------|---------|
| 产能数据 | 提示分该项不计分，标注「数据缺失」 |
| 上下游关系 | 不进入 Serenity 路径，待专家补充 |
| 机构覆盖数 | 用成交额分位替代 |
| 研报 | 仅依赖图谱事实，报告标注「证据不足」 |

> **原则：宁可缺数据，不可编造数据。** 缺失一律显式降级并在前端标注，禁止用估算值静默填充。

## 6. 数据接入架构

```
外部 API / 文件
    ↓
接入适配器 (adapter)
    ↓
原始层 (MinIO + PostgreSQL raw)
    ↓
清洗任务 (Celery)
    ↓
标准层 (PostgreSQL)
    ↓
知识抽取 / 图谱同步
```

### 6.1 PostgreSQL 数据模型

原始数据层（ODS）：

```
ods_financial_statement    # 财报原始
ods_announcement           # 公告元数据 + MinIO 路径
ods_industry_metric        # 产业指标时序
ods_market_daily           # 日度行情
ods_research_report        # 研报元数据
```

知识映射层：

```
entity_alias               # 实体对齐（见 02 知识工程 §5.3）
data_source_registry       # 数据源注册
field_lineage              # 字段 → 来源 → 更新时间
quality_flag               # 数据质量标记
```

### 6.2 字段血缘示例（field_lineage）

```yaml
field: product.expansion_cycle_months
sources:
  - source: announcement
    ref: "2025-08-15 扩产公告"
    value: 24
    fetched_at: 2026-06-01
  - source: research_report
    ref: "XX证券深度报告 p.15"
    value: 18
    fetched_at: 2026-05-20
resolved_value: 24
resolution_rule: "announcement 优先"
```

### 6.3 更新调度

| 任务 | 频率 | 执行方式 | 失败降级 |
|------|------|---------|---------|
| 公告同步 | 日 | Celery | 跳过，告警 |
| 行情 / 估值 | 日 | Celery | 用 T-1 数据 |
| 产业指标 | 周 / 月 | Celery + 人工 | 标记 stale |
| 图谱增量 | 日 | 抽取 pipeline | 进审核队列 |
| 瓶颈提示分重算 | 周 | 规则引擎 | 保留上次分数 |

## 7. 一期数据范围（MVP）

**聚焦赛道：AI 算力（含光模块、先进封装）**

最小可行数据源（控制成本）：

| 类别 | 一期方案 |
|------|---------|
| 行情 / 估值 / 财报 | 1 家主流数据商 API（Wind 或 iFinD 二选一） |
| 公告 | 巨潮免费接口 + 爬虫备份 |
| 研报 | 内部已有 PDF + 手动上传 |
| 产业数据 | 人工维护 Excel 模板 + 定期导入 |
| 供应链 | 公告抽取 + 研究员手工补全 |

一期数据量目标：

| 数据 | 一期目标 |
|------|---------|
| 产业链产品节点 | ≥ 50 个（已确认） |
| 上市公司 | ≥ 30 家 |
| 已确认关系 | ≥ 100 条 |
| 研报入库 | ≥ 200 份 |
| 公告 | 覆盖池内公司 2 年 |

## 8. 数据质量 SLA

| 指标 | 目标 |
|------|------|
| 行情数据延迟 | < 1 交易日 |
| 公告入库延迟 | < 4 小时 |
| 图谱 confirmed 数据占比 | > 80% |
| 关键属性双源验证率 | > 60% |
