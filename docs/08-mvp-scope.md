# 一期 MVP 范围（历史参考）

> **v2.0 起请以 [08-implementation-phases.md](./08-implementation-phases.md) 与 DESIGN.md §11 为准。**

## 1. MVP 目标

在 **单赛道（AI 算力）** 上跑通端到端闭环：

```
数据接入 → 知识抽取 → 专家校准 → 图谱展示 → 双逻辑候选 → GraphRAG 报告 → 人工入池
```

## 2. 范围边界

### 2.1 做（In Scope）

| 模块 | 能力 |
|------|------|
| 本体 | 核心类 Product/Company/Sector/Event，AI 算力子本体 |
| 图谱 | Neo4j 存储、G6 可视化、链条校准 UI |
| 知识抽取 | 公告 + 研报 LLM 三元组抽取，draft → confirmed 流程 |
| 溯源 | 属性级溯源展示 |
| 买方逻辑 | 规则提示分 + 人工确认瓶颈 |
| Serenity | 逆向 3–4 跳 + 剪枝 + 候选列表 |
| GraphRAG | 事实推理 + 反证 + 报告草稿 |
| 人机协同 | 入池确认、覆盖留痕、报告审核 |
| 看板 | 核心产业指标（产能、价格、毛利率） |

### 2.2 不做（Out of Scope）

| 模块 | 延后至 |
|------|--------|
| GNN 打分 | 三期或不做 |
| 全行业图谱 | 二期起逐步扩展 |
| Flink 实时流 | 不做，用 Celery |
| NebulaGraph | 不做，单 Neo4j |
| 自动交易 / 回测 | 不做 |
| 移动端 | 三期 |
| LLM 微调 | 二期评估 |

## 3. 里程碑

### M1（第 1 月）：基座

- [ ] 项目骨架、Docker Compose
- [ ] PostgreSQL + Neo4j + Redis 就绪
- [ ] OWL 本体 v1.0
- [ ] AI 算力赛道手工种子图谱（30+ 节点）
- [ ] 图谱可视化只读页面

### M2（第 2 月）：知识流水线

- [ ] 公告/研报接入
- [ ] LLM 抽取 + draft 入库
- [ ] 专家校准后台
- [ ] 溯源展示
- [ ] 规则提示分引擎

### M3（第 3 月）：业务闭环

- [ ] Serenity 逆向溯源
- [ ] 双逻辑候选池 + 入池确认
- [ ] GraphRAG 报告草稿 + 审核
- [ ] 产业指标看板
- [ ] 历史案例复盘验证（光模块 2023）

## 4. 验收标准

| 验收项 | 标准 |
|--------|------|
| 端到端演示 | 从赛道选择到入池确认全流程可演示 |
| 逻辑复现 | 光模块/CoWoS 案例逻辑链可复现 |
| 溯源 | 报告关键论断 100% 可溯源 |
| 人工门控 | 无人工确认不得入正式池 |
| 性能 | 图谱 3 跳查询 < 2s |
