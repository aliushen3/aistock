# 双投研逻辑融合标准（定稿版）

## 1. 逻辑定位

两套逻辑在同一产业链上提供**两种视角、两种收益来源**，通过知识图谱统一表示，经不同推理路径产出不同候选池。

| 维度 | 专业买方逻辑 | Serenity 逆向逻辑 |
|------|-------------|-------------------|
| 推理方向 | 自上而下：赛道 → 全链拓扑 → 瓶颈环节 → 龙头 | 自下而上：终端景气 → 向上 3–4 层 → 小众环节 |
| 收益来源 | Beta 主升浪 + Alpha 壁垒溢价 | 低估弹性、认知差修复 |
| 标的偏好 | 壁垒龙头、业绩兑现、机构覆盖适中 | 中小市值、低拥挤、低覆盖 |
| 持仓周期 | 中长期（6–24 月） | 中短期（3–12 月） |
| 主要风险 | 估值透支、景气见顶 | 流动性差、逻辑未被认知 |

## 2. 专业买方逻辑 — 可执行标准

### 2.1 赛道 Beta 识别（定性为主）

**必要条件（需研究员确认「高景气」标签）：**

- 下游需求复合增速 > 20%（数据提示，非硬性）
- 行业资本开支同比为正
- 至少 2 份独立研报支持景气判断（证据引用）

**系统动作：** 在赛道节点打 `beta_candidate` 标签，进入拓扑构建流程。

### 2.2 全产业链拓扑

- 构建 终端 → 中游 → 上游材料/耗材 完整 DAG
- 每条边标注：成本占比、传导时滞、替代关系
- 分叉路径保留，不强行合并

### 2.3 瓶颈环节识别（规则提示 + 人工确认）

**规则提示条件（满足 ≥3 项则打 `bottleneck_hint`）：**

| 条件 | 阈值 |
|------|------|
| 扩产周期 | > 18 个月 |
| 客户认证周期 | ≥ 2 年 |
| 行业 CR4 | > 60% |
| 海外设备/材料依赖 | 是 |
| 短期产能缺口 | 需求增速 > 产能增速 |
| 涨价持续性 | 连续 2 季度以上 |

**人工确认后** 升级为 `bottleneck_confirmed`，写入知识库并记录确认人与时间。

### 2.4 标的映射

- 公司节点通过 `PRODUCES` 关系关联瓶颈产品
- 定性筛选：毛利率趋势、业绩兑现、客户结构
- 输出：**买方专业候选池**

## 3. Serenity 逆向逻辑 — 可执行标准

### 3.1 输入

与买方共用「已确认高景气」赛道库（`beta_confirmed` 标签）。

### 3.2 图遍历算法（见 `05-serenity-algorithm.md`）

- 从终端产品节点出发，沿 `UPSTREAM_OF` 反向遍历 3–4 跳
- 剪枝与排序规则见算法文档

### 3.3 小众咽喉环节判定

**`serenity_niche` 标签条件（满足 ≥4 项）：**

| 条件 | 阈值 |
|------|------|
| 产品层级 | 二级/三级耗材或材料 |
| 成本占比 | < 5% 但不可替代 |
| 替代难度 | 高（本体属性） |
| 机构覆盖 | < 5 家 |
| 总市值 | < 200 亿（可配置） |
| 成交额分位 | < 30% |
| 终端龙头排除 | 非赛道市值前 3 |

### 3.4 输出

**Serenity 逆向候选池**，附带逆向路径可视化。

## 4. 三种融合模式

### 模式 A：买方专业模式

- 路径：Beta 赛道 → 拓扑 → 瓶颈确认 → 龙头映射
- 输出：`pool_buy_side`
- 适用：稳健主升、机构风格

### 模式 B：Serenity 逆向模式

- 路径：终端赛道 → 逆向溯源 → 小众筛选
- 输出：`pool_serenity`
- 适用：挖掘低估弹性

### 模式 C：双逻辑融合模式

**交集策略（高优先级）：**

```
同一赛道 AND (
  公司关联产品.bottleneck_confirmed = true
  OR 公司关联产品.serenity_niche = true
)
AND 两池均通过 LLM 反证检查
```

**互补策略（组合建议）：**

- 龙头（买方池）+ 小众配套（Serenity 池）
- 系统生成「核心 + 弹性」组合草稿，**不自动分配仓位**

### 模式优先级（人工审批前排序）

| 优先级 | 条件 | 含义 |
|--------|------|------|
| P0 | 同一公司同时出现在买方池与 Serenity 池 | 双逻辑共振 |
| P1 | 同一赛道，买方为龙头 + Serenity 为小众配套 | 组合建议 |
| P2 | 仅满足单一逻辑 | 普通候选 |

### 4.4 与量化打分的关系

| 环节 | 量化角色 |
|------|---------|
| 瓶颈提示分 | 辅助排序候选产品，不自动标「确认瓶颈」 |
| 拥挤度分位 | 辅助 Serenity 排序，不自动剔除 |
| 估值分位 | 风险提示，不自动剔除 |
| 最终入池 | **仅人工确认** |

## 5. 定性优先的决策流程

```
Step1  识别赛道 Beta          → 研究员确认景气
Step2  构建产业拓扑图谱        → 专家校准链条
Step3  瓶颈规则提示            → 研究员确认/否决
Step4  Serenity 逆向溯源       → 研究员筛选小众环节
Step5  GraphRAG 逻辑验证+反证  → 研究员审核草稿
Step6  分层候选池              → 研究员确认入池
Step7  动态跟踪                → 事件触发复核
```

**与原方案差异：** Step3 由「GNN 自动打分」改为「规则提示 + 人工确认」；Step6 由「自动输出」改为「人工确认入池」。

## 6. 本体约束规则（OWL 表达要点）

```turtle
# 伪代码示意
:BottleneckHint a owl:Class ;
  owl:equivalentClass [
    owl:intersectionOf (
      :Product
      [ owl:onProperty :expansionCycleMonths ; owl:someValuesFrom [ owl:minInclusive 18 ] ]
      [ owl:onProperty :cr4Concentration ; owl:someValuesFrom [ owl:minInclusive 0.6 ] ]
    )
  ] .

:SerenityNiche a owl:Class ;
  owl:equivalentClass [
    owl:intersectionOf (
      :Product
      [ owl:onProperty :costRatio ; owl:someValuesFrom [ owl:maxInclusive 0.05 ] ]
      [ owl:onProperty :substitutionDifficulty ; owl:hasValue "high" ]
    )
  ] .
```
