# 系统定位：AI Native 知识驱动定性投研操作系统

> v2.0 修订。冲突以 [DESIGN.md](./DESIGN.md) 为准。

## 1. 一句话定位

**本系统是「AI Native 知识驱动定性投研操作系统（IR-OS）」**——以产业链知识图谱与 Palantir 式 Ontology 为确定性基座，以大模型与多智能体为推理编排引擎，辅助研究员完成赛道发现、瓶颈识别、标的筛选与逻辑论证。

> **量化打分仅作辅助排序；智能体产出均为提案，须经人工 Ontology Action 确认。**

## 2. 不是什么

| 类型 | 说明 |
|------|------|
| 量化交易系统 | 不做因子回测、不做自动下单、不以历史收益率为优化目标 |
| 黑盒荐股工具 | 所有结论必须附带证据链与推理路径，可审计、可反驳 |
| 全自动选股机 | 最终入池、仓位、买卖时点由人工研究员确认 |
| 行情预测系统 | 不承诺收益，不输出买卖时点信号 |

## 2.1 是什么

| 能力 | 说明 |
|------|------|
| 知识资产平台 | 沉淀产业链拓扑、瓶颈判定、证据链、专家校准记录 |
| 定性推理助手 | 辅助完成 Beta 赛道识别、Alpha 环节锚定、逆向溯源、风险反证 |
| 协作投研工作台 | 支持研究员批注、复核、覆盖、投委会材料导出 |
| **AI Native 智能体层** | Agent 矩阵（A 类真 LLM / B 类确定性 Pipeline）+ ReAct；主动扫描、起草、对抗、告警；人终审 |

## 2.2 三条设计主线（v3.0 核心）

横切 L0–L6 的三条方法论纪律，对应买方真正赚钱（或亏钱）的关键环节。对齐主册 [DESIGN.md §1.4](./DESIGN.md)。

| 主线 | 一句话 | 落点 |
|------|--------|------|
| **反证优先** | 多空等强对抗，空头论点未回应不得入池 | 独立 BearCaseAgent + 入池硬闸门 |
| **知识保鲜** | 每条结论有半衰期，过期自动降级；回答「现在还成立吗」 | 保鲜状态机 + 瓶颈生命周期 |
| **预期差与价值捕获** | 识别瓶颈 ≠ 存在超额；入池前必答 price-in 与利润捕获 | 入池三道闸 |

> **三句话纪律**：① 识别瓶颈 ≠ 超额，入池必答预期差与价值捕获；② 任何看多必须配等强看空，未回应不得入池；③ 任何结论都有保鲜期，过期降级。

## 4. 定性 vs 量化：边界约定

### 4.1 辅助量化层（L1）允许做的事

- **提示分（Hint Score）**：0–100 分，表示「该环节具备瓶颈特征的置信提示」，非投资评分
- **排序辅助**：在同赛道候选中，按提示分、拥挤度、估值分位等做**初步排序**
- **阈值提醒**：如 CR4>60%、扩产周期>18 月等，以**标签/告警**形式呈现，非硬性过滤（除非研究员启用）
- **可视化编码**：图谱节点颜色、看板红黄绿，表达「值得关注程度」

### 4.2 辅助量化层（L1）禁止做的事

- 禁止仅凭分数自动入池或出池
- 禁止输出「买入/卖出」指令
- 禁止将 GNN/机器学习模型作为一期核心路径（见 MVP 文档）
- 禁止隐藏打分权重与数据来源

### 4.3 定性推理层（L3）必须做的事

- 输出完整**投资逻辑链**：赛道 Beta → 瓶颈环节 → 标的映射 → 风险反证
- 每条关键判断附带**证据引用**（公告、研报页码、图谱关系）
- 支持研究员**批注、覆盖、否决**，并记录理由
- 支持**情景分析**：景气延续 / 供需缓解 / 技术替代 三种情景下的逻辑变化

## 5. 原方案修订对照

| 原方案表述 | 修订后 |
|-----------|--------|
| GNN 图神经网络瓶颈 Alpha 量化打分 | **规则引擎 + 图谱查询** 生成「瓶颈提示分」，一期不引入 GNN |
| 得分≥70 为核心瓶颈 Alpha 节点 | ≥70 为「高提示」标签，需研究员定性确认后方可标为「已确认瓶颈」 |
| 实现标准化、自动化选股 | 实现标准化、**辅助式**选股流程，**人工确认**为必经环节 |
| Flink 实时流 | Celery 定时任务 + 事件触发增量更新 |
| NebulaGraph + Neo4j 双图库 | 一期仅 **Neo4j** |
| 全行业产业链图谱 | 一期 **1–2 条核心赛道**（见 MVP 文档） |

## 6. 系统输出物定义

| 输出物 | 性质 | 是否需人工确认 |
|--------|------|----------------|
| 产业链拓扑图 | 知识资产 | 新关系需专家校准 |
| 瓶颈提示分 | 辅助量化 | 否（需标注置信度 + 保鲜状态） |
| 知识保鲜状态 | 知识治理 | 过期自动降级，stale 需复核 |
| 候选标的池 | 辅助清单 | **是**，入池前必过三道闸 |
| 预期差/价值捕获评估 | 定性推理 | 入池必答 |
| 看空论点（Bear Case） | 定性推理 | **是**，高severity 未回应阻断入池 |
| 投研逻辑草稿（看多） | 定性推理 | **是**，发布前必须审核 |
| 产业跟踪看板 | 数据展示 | 异常指标需人工研判 |

## 7. 设计原则（开发约束）

1. **证据优先**：无证据不生成结论
2. **人机协同**：机器起草，人工定稿
3. **知识可沉淀**：每次人工修正必须回写知识库
4. **反证等强**：空头论点与多头论点同等投入，未被正面回应不得入池
5. **知识有时效**：每条结论标注保鲜状态，过期自动降级
6. **识别 ≠ 赚钱**：入池前必答预期差与价值捕获
7. **非量化优先**：新功能先问「能否用图谱/本体/案例推理实现」，再考虑是否加辅助分数

## 8. 提示分输出规范（Score Card）

所有量化提示分以统一结构输出，**自带证据引用与免责标记**，禁止裸分数：

```yaml
score_card:
  entity_id: "product:coated_copper_foil"
  score_type: "bottleneck_alpha_hint"   # 提示分，非决策分
  value: 72
  weight_breakdown:
    supply_rigidity:   { score: 22, max: 30, evidence_refs: ["ev-001", "ev-002"] }
    tech_barrier:      { score: 18, max: 25, evidence_refs: ["ev-003"] }
    supply_demand_gap: { score: 20, max: 25, evidence_refs: ["ev-004"] }
    concentration:     { score: 12, max: 20, evidence_refs: ["ev-005"] }
  label_hint: "疑似瓶颈环节"             # 提示标签，非最终判定
  requires_human_review: true
  disclaimer: "本分数为辅助排序提示，不构成投资建议"
```

## 9. 合规与免责（技术实现要求）

- 所有页面底部固定展示：「本系统输出仅为投研参考，不构成投资建议」
- 报告导出自动附带：生成时间、知识版本号、参与复核人、证据完整度
- 审计日志保留：谁、何时、对哪条结论做了什么操作（确认 / 否决 / 覆盖）
