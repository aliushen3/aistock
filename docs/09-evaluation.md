# 评估与验证方案

## 1. 评估原则

本系统**不以历史收益率**为主要评估指标，而采用**知识质量 + 逻辑复现 + 人机协同效率 + 结果回溯**四维评估（v3.0 新增第 4 维）。对齐主册 [DESIGN.md §11.1](./DESIGN.md)。

> **排除「优化收益」≠ 排除「回溯对错」**：不追求 Sharpe/胜率，但须 ex-post 追踪判断的事后正确性，用于校准与信誉。

## 2. 评估维度

### 2.1 知识质量评估

| 指标 | 方法 | 频率 |
|------|------|------|
| 三元组准确率 | 随机抽检 100 条，专家打分 | 每月 |
| 链条完整度 | 终端到上游路径覆盖率 | 每月 |
| 溯源覆盖率 | 展示属性有溯源比例 | 每周自动 |
| 冲突解决时效 | SLA 内完成率 | 每周 |

### 2.2 逻辑复现评估（定性案例）

**一期必选案例：**

| 案例 | 验证问题 |
|------|---------|
| 光模块 2023 主升浪 | 系统能否识别 EML/硅光等瓶颈环节 |
| CoWoS 先进封装 | 逆向溯源能否找到小众材料/设备 |
| 失败案例：某扩产后瓶颈缓解 | 反证推理能否提前提示风险 |

**评估方法：**

1. 研究员独立写逻辑链（金标准）
2. 系统生成逻辑链
3. 双人盲评对比：节点覆盖率、关键环节命中、风险识别
4. 评分 1–5，目标均值 ≥ 4

> **盲测要求（v3.0，防后视偏差）**：历史案例的逻辑复现存在**幸存者与后视偏差**（本体已知答案）。复盘须用 **T 时点之前的数据**回放，禁止把结果信息喂给系统；评估「不知道答案时能否复现关键环节」，而非「知道答案后能否自圆其说」。

### 2.3 人机协同效率

| 指标 | 目标 |
|------|------|
| 报告草稿采纳率（修改 < 30% 即可发布） | ≥ 50% |
| 候选池命中率（入池标的 3 月内逻辑仍成立） | ≥ 70% |
| 研究员校准工时/赛道 | 下降趋势 |

### 2.4 结果回溯（Outcome Tracking，v3.0 新增）

**不追求 Sharpe/胜率**，但 ex-post 追踪判断的事后正确性：

| 指标 | 定义 |
|------|------|
| 瓶颈兑现率 | confirmed 瓶颈中事后真兑现（持续涨价/盈利）的比例 |
| 假动作率 | confirmed 瓶颈中很快被证伪/缓解的比例 |
| 空头应验率 | 高severity 空头论点事后应验的比例 |

**结果回流**：① 提示分权重校准（见 06 §7）；② 分析师与 scorer 信誉记录；③ 案例库扩充。

### 2.5 系统不可用指标（明确排除）

- 年化收益率、夏普比率
- 自动回测胜率
- 与指数超额对比

## 4. LLM 输出评估

| 检查项 | 自动/人工 |
|--------|----------|
| Citation 存在率 | 自动 |
| 数值与图谱一致 | 自动 |
| 逻辑链完整性 | 人工 |
| 反证覆盖 | 人工 |
| 无幻觉断言 | 人工抽检 |

## 5. 灰度发布

| 阶段 | 用户 | 范围 |
|------|------|------|
| Alpha | 开发 + 1 名研究员 | 单赛道只读 |
| Beta | 投研小组 | 校准 + 入池 |
| GA | 全投研团队 | 全功能 |

## 6. 持续改进闭环

```
评估结果 → 问题分类 → 本体/规则/Prompt 修正 → 版本发布 → 再评估
```
